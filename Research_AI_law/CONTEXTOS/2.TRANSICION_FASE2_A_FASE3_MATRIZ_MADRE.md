## 3. Que es la Matriz Madre

La Matriz Madre sera la tabla base que alimentara la EDA principal, el feature engineering y el modelamiento posterior.

Definicion operativa:

- Cada fila representa un pais o entidad comparable.
- Cada columna representa un atributo observado o derivado bajo una regla documentada.
- Cada celda contiene un valor real extraido desde una fuente, o un valor faltante explicito.
- Cada valor debe conservar trazabilidad hacia archivo, hoja/tabla, variable original, pais/entidad, ano y regla de transformacion.

La Matriz Madre no es solo un CSV ancho. Debe ser un sistema de datos compuesto por:

1. Una tabla principal pais/entidad x atributo.
2. Un diccionario de variables.
3. Una tabla de trazabilidad celda/fuente.
4. Un registro de decisiones metodologicas.
5. Un reporte de validacion humana.

Archivo de referencia:

```text

```

Hoja principal de referencia:

```text
Matriz MADRE
```

La hoja `Matriz MADRE` del ejemplo representa una matriz pais x atributo ya avanzada. Tiene:

- una fila por pais;
- columnas identificadoras como `iso3`, `country_name`, `region`, `income_group`;
- bloques de variables por fuente o tema;
- columnas de regulacion IA;
- columnas de preparacion gubernamental Oxford;
- columnas de innovacion WIPO;
- columnas de gobernanza / desarrollo / controles;
- una logica de matriz densa o subconjunto con alta cobertura;
- un diccionario de variables en hoja separada;
- hojas adicionales por fuente o bloque de evidencia.

Este archivo es util como referencia de formato final, pero no debe copiarse mecanicamente.

### 3.1.1 Que si conviene adoptar del ejemplo

La Fase 3 debe adoptar estas ideas del Excel de ejemplo:

- Una Matriz Madre wide para lectura humana, con una fila por pais.
- Columnas ordenadas por bloques tematicos.
- Prefijos de fuente o bloque en los nombres de variables.
- Hoja de diccionario de variables.
- Hoja de fuentes.
- Separacion entre matriz completa y matriz densa candidata.
- Marcado explicito de `NA` como dato no disponible.
- Metadata por variable: fuente, ano, tipo, escala, completitud.
- Informacion de fila: cobertura, inclusion/exclusion, comentarios.

### 3.1.2 Que no conviene adelantar todavia

La Fase 3 no debe adelantar decisiones propias de EDA final, feature engineering o modelamiento. Por eso, del ejemplo no se debe copiar automaticamente:

- Variables llamadas "supervivientes" sin una regla documentada.
- La muestra de 86 paises como universo automatico.
- La matriz densa como muestra analitica final.
- Codificaciones propias sin trazabilidad completa.
- Indices compuestos nuevos sin formula auditada.
- Clasificacion definitiva de variables como Y, X1 o X2.
- Estimaciones marcadas como pendientes de reemplazo por datos verificados.
- Variables de "gap" si aun no fueron extraidas desde fuentes reales.

El ejemplo contiene elementos que parecen pertenecer a fases posteriores, especialmente:

- seleccion de variables por cobertura;
- construccion de submatriz densa;
- variables compuestas de intensidad regulatoria;
- posibles estimaciones o placeholders que deben reemplazarse por tablas verificadas;
- organizacion orientada a modelamiento.

Estas decisiones pueden ser validas mas adelante, pero solo despues de que la Fase 3 haya construido una base auditable.

### 3.1.3 Opinion metodologica recomendada

La ruta profesional recomendada es construir tres capas, en este orden:

1. **Capa auditora larga**: una tabla long donde cada fila sea una observacion fuente-variable-pais-ano. Esta es la fuente de verdad.
2. **Matriz Madre completa wide**: una tabla pais x atributo, derivada desde la capa larga mediante reglas documentadas.
3. **Matriz densa candidata**: subconjunto de paises y variables con cobertura suficiente, usada para EDA principal y modelamiento preliminar.

La hoja `Matriz MADRE` del Excel ejemplo corresponde principalmente a la capa 2, con algunos elementos de capa 3. Por lo tanto:

- Si se intenta construir directamente una matriz como el ejemplo, se corre el riesgo de perder trazabilidad.
- Si se construye primero una matriz larga auditable, despues es seguro producir una matriz wide como el ejemplo.
- La matriz densa debe considerarse un output derivado, no la fuente de verdad.

Decision recomendada:

> Usar `Matriz_EJEMPLO.xlsx` como patron de entrega final human-readable, pero no como punto de partida tecnico. El punto de partida tecnico debe ser una matriz larga con trazabilidad completa.

### 3.1.5 Distincion clave: Matriz Madre vs EDA final

La Matriz Madre no debe responder todavia la pregunta sustantiva del estudio.

En Fase 3 corresponde:

- integrar datos;
- limpiar llaves;
- documentar reglas;
- construir variables observadas o transformaciones minimas;
- dejar trazabilidad;
- preparar una tabla apta para EDA principal.

En EDA final o fases posteriores corresponde:

- analizar distribuciones sustantivas;
- comparar grupos regulatorios;
- seleccionar outcomes y tratamientos;
- construir indices finales;
- evaluar correlaciones;
- definir submuestras analiticas;
- preparar modelos.

Por lo tanto, crear una matriz wide como el ejemplo no es adelantarse si se hace como producto de integracion y auditoria. Si esa matriz ya decide variables supervivientes, tratamientos, outcomes o muestra final para modelar, entonces si se estaria adelantando.

Regla practica:

> Fase 3 puede construir la forma de la matriz. No debe cerrar todavia la interpretacion causal ni la estrategia final de modelamiento.

## 4. Principios obligatorios para Fase 3

### 4.1 Cero datos sinteticos

No se permite:

- Inventar valores.
- Completar datos por intuicion.
- Copiar valores desde resumenes no trazables.
- Rellenar paises no cubiertos por una fuente.
- Convertir ausencia de dato en cero, salvo que la fuente lo defina explicitamente.

### 4.2 Human in the loop

Toda decision critica debe tener revision humana:

- Crosswalk de nombres de pais a ISO3.
- Inclusion/exclusion de territorios.
- Tratamiento de agregados regionales.
- Seleccion de variables candidatas a Matriz Madre.
- Eleccion de snapshot temporal.
- Uso de rankings.
- Reglas de transformacion.

### 4.3 Trazabilidad de celda

Cada celda de la Matriz Madre debe poder responder:

- De que fuente viene?
- De que archivo exacto?
- De que hoja o tabla?
- De que variable original?
- De que pais o entidad?
- De que ano o periodo?
- Que regla se aplico?
- Quien valido la regla?
- Si falta, falta porque no existe, no aplica o no se pudo mapear?

### 4.4 Separacion entre dato observado y decision metodologica

Ejemplo:

- Dato observado: Oxford tiene `total_score` para Chile en 2025.
- Decision metodologica: usar 2025 como snapshot.

Ambas cosas deben documentarse en lugares distintos.

- Crear reglas explicitas para no aplica.
