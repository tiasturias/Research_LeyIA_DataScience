```

ecosystem_outcome           вҶ’ Oxford readiness, Stanford economy, WIPO innovation outputs
```

**Importante:** los bloques NO son Y / X1 / X2. Son una taxonomГӯa de auditorГӯa que permite calcular cobertura por dimensiГіn conceptual y serГЎ insumo de Fase 4-6 para discutir candidatos a outcome y tratamiento, pero la decisiГіn final pertenece a esas fases.

## 5. Esquemas de datos formales (pandera)

### 5.1 `MatrizLargaPanelSchema`

Grano: una observaciГіn = `(iso3, source_id, original_variable, year)`.

| Campo | Tipo | ValidaciГіn |
|---|---|---|
| `cell_id` | str (PK) | `<iso3>_<source>_<var>_<year>`, Гәnico |
| `iso3` | str | Match contra ISO3 canГіnico |
| `country_name_canonical` | str | Not null |
| `entity_type` | enum | `country_iso3` \| `territory_iso3` \| `region` \| `subnational` \| `organization_or_group` \| `global` |
| `source_id` | enum | `iapp` \| `oxford` \| `wb` \| `wipo` \| `microsoft` \| `stanford` \| `oecd` \| `anthropic` |
| `table_id` | str | FK a `fase3_tablas_seleccionadas.csv` |
| `original_variable` | str | Not null |
| `variable_matriz` | str | FK a `fase3_diccionario_variables.csv` |
| `year` | int \| None | Rango [2018, 2026] o NaN si cross-section |
| `value_original` | str | Raw como string preserva el valor exacto |
| `value_numeric` | float \| NaN | NaN si no aplica |
| `value_text` | str \| None | Para categГіricas/texto |
| `unit` | str | `pct`, `score_0_100`, `usd`, `count`, `rank`, `wgi_-2.5_2.5`, `binary`, `categorical`, `text` |
| `direction` | enum | `higher_better` \| `lower_better` \| `categorical` \| `unknown` |
| `confidence_level` | enum | `verified` \| `extracted` \| `inferred` \| `missing_explicit` \| `missing_no_data` |
| `extraction_rule` | str | Regla aplicada (ej: `"raw_value"`, `"sum_subindicators"`) |
| `source_file` | str | Path relativo |
| `source_sheet` | str \| None | Hoja XLSX si aplica |
| `row_identifier` | str | Identificador Гәnico en archivo fuente para auditorГӯa |
| `created_at` | datetime | UTC ISO |
| `extractor_version` | str | VersiГіn del mГіdulo extractor |

### 5.2 `MatrizLargaSnapshotSchema`

Igual a Panel + campos: `snapshot_rule`, `year_used`, `years_collapsed` (lista). Se construye desde Panel aplicando `temporal_rules.yaml`.

### 5.3 `MatrizWideSchema`

Una fila por `iso3`. Columnas:
- Identificadoras: `iso3`, `country_name`, `entity_type`, `region`, `income_group`
- Cobertura: `n_sources_present`, `source_list`, `included_in_dense_candidate`
- Variables: una por entrada del diccionario, prefijada por fuente (`iapp_*`, `oxford_*`, `wb_*`, `wipo_*`, `ms_*`, `stanford_*`, `oecd_*`, `anthropic_*`)
- Sufijos auxiliares por variable: `*_year_used`, `*_confidence`

### 5.4 `DiccionarioVariablesSchema`

PK: `variable_matriz`. Campos: `source_id`, `table_id`, `original_variable`, `tipo_matriz`, `unit`, `direction`, `bloque_tematico` (enum de 6 bloques), `regla_temporal_default`, `is_primary` (bool вҖ” indica si es primaria cuando hay redundancia entre fuentes), `redundant_with` (lista), `notes`.

## 11. Riesgos y mitigaciones

| Riesgo | MitigaciГіn |
|---|---|
| Crosswalk Microsoft tarda mucho | Fuzzy matching automГЎtico + revisiГіn humana en widget; ~76 paГӯses Г— 30s вүҲ 40 min realistas |
| Variable redundante mal asignada | `is_primary` + `redundant_with` explГӯcitos en diccionario; tests verifican 1 sola primaria por concepto |
| Snapshot pierde varianza temporal | Mantenemos siempre `matriz_larga_panel.csv` para reaplicar reglas |
| Notebook se rompe en otro entorno | LГіgica en `src/`, notebook solo invoca; tests en CI verifican |
| Decisiones no quedan registradas | `decisions.yaml` + `human_review_log.csv` como single source of truth |
| Fase 4 quiere modificar la matriz | API estable + inmutabilidad por versiГіn; cambios producen v1.1, no se sobreescribe v1.0 |
| Variables categГіricas se confunden con numГ©ricas | Pandera valida `tipo_matriz` y rangos por unit |
| PГ©rdida de trazabilidad al pivotear | `matriz_madre_trazabilidad.csv` mapea cada cell_id_wide вҶ’ cell_id_largo |
