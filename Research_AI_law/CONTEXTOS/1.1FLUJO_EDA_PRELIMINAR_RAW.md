## 2. Visión general del pipeline

El EDA Preliminar RAW es una auditoría exploratoria neutral de las 8 fuentes unificadas disponibles. No es análisis causal, no es modelamiento, no es la matriz madre. Es el paso previo necesario para construir la matriz madre de forma defensible.

### 2.1 Flujo de alto nivel

```text
                    DATOS FUENTE (8 archivos unificados)
                                    │
                    ┌───────────────┼───────────────┐
                    │                               │
            FASE A: AUDITORÍA               FASE B: INTEGRACIÓN
            INDIVIDUAL                      CROSS-SOURCE
            (una fuente a la vez)           (todas las fuentes juntas)
                    │                               │
                    ├─ Inventario fuentes           ├─ Geografía comparada
                    ├─ Inventario tablas            ├─ Diagnóstico calidad
                    ├─ Auditoría esquema            ├─ Matriz país × fuente
                    ├─ Temporalidad                 ├─ Matriz país × atributo
                    ├─ Completitud                  ├─ Recomendaciones
                    ├─ Perfil univariado            └─ Resumen ejecutivo
                    │
                    ▼
            11 ARCHIVOS DE OUTPUT + 1 NOTEBOOK
                    │
                    ▼
            FASE 3: PREPARACIÓN DE DATOS / MATRIZ MADRE
```

### 2.2 ¿Por qué dos fases?

Porque no se puede integrar lo que no se entiende. Primero hay que entender cada fuente por separado (qué mide, qué cubre, qué le falta), y solo después se puede decidir cómo unirlas.

## 3. Fase A — Auditoría individual por fuente

En esta fase se examina cada fuente de manera aislada, sin mezclar información entre fuentes.

### 3.1 ¿Qué se hace por fuente?

Para cada una de las 8 fuentes se ejecutan 5 auditorías:

| Auditoría | Pregunta que responde | Output |
|---|---|---|
| **Inventario de tablas** | ¿Qué hojas/archivos tiene esta fuente? | Filas en `inventario_tablas.csv` |
| **Auditoría de esquema** | ¿Qué columnas tiene cada tabla? ¿Qué tipos son? ¿Cuántos nulos? | Filas en `inventario_variables.csv` |
| **Temporalidad** | ¿Es cross-section, panel, wide, long? ¿Qué años cubre? | Fila en `cobertura_temporal_fuente.csv` |
| **Completitud** | ¿Qué porcentaje de datos está poblado por variable y por país? | Filas en `cobertura_pais_atributo.csv` |
| **Perfil univariado** | ¿Cuáles son los estadísticos de cada variable numérica/categórica? | Resumen inline en notebook |

```text

Auditoría de tablas:

  - Detalle_2019: 194 filas × 34 columnas → AUXILIAR
  - Diccionario_Variables: 126 filas × 9 columnas → DICCIONARIO
  - ISO3_Mapping: 229 filas × 11 columnas → AUXILIAR

  - iso3: identifier, 195 únicos, 0% nulos
  - pais_original: geo_identifier, 229 únicos, 0% nulos (inconsistente)
  - year: time, 7 valores únicos (2019-2025), 0% nulos
  - total_score: numeric_continuous, 1,283 no nulos, 0% nulos
  - [17 columnas]: empty, 0% no nulos → PROBLEMA

Temporalidad:
  - Tipo: panel_long (pais-año)
  - Años: 2019-2025
  - PROBLEMA: 3 regímenes metodológicos distintos
    - 2019: escala 0-10, framework 4_clusters

  - Promedio: ~30% (muy variable)
  - 17 columnas 100% vacías
  - Variables 2019: solo 15% de cobertura (solo 194 de 1,283 filas)
```

### 3.3 ¿Qué NO se hace en la auditoría individual?

- NO se comparan variables entre fuentes
- NO se deciden qué variables entran a la matriz madre
- NO se imputan datos
- NO se codifican categorías
- NO se crean snapshots definitivos

## 4. Fase B — Integración cross-source

Una vez que cada fuente está auditada individualmente, se cruzan los resultados para entender cómo encajan entre sí.

### 4.1 ¿Qué se hace en integración?

| Auditoría | Pregunta que responde | Output |
|---|---|---|
| **Geografía comparada** | ¿Qué países aparecen en qué fuentes? ¿Quién está en todas? ¿Quién solo en una? | `cobertura_pais_fuente.csv` + `entidades_no_pais_o_sin_iso3.csv` |
| **Diagnóstico de calidad** | ¿Cuáles son TODOS los problemas detectados en TODAS las fuentes? | `data_quality_issues.csv` |
| **Matriz país × fuente** | ¿Qué tan integrable es cada país? | Matriz de integrabilidad |
| **Matriz país × atributo** | ¿Qué tan completos son los datos por país × variable? | Matriz de auditoría amplia |
| **Recomendaciones** | ¿Qué decisiones debe tomar la fase de wrangling? | `recomendaciones_wrangling.csv` |
| **Resumen ejecutivo** | ¿Cuál es el panorama completo en lenguaje humano? | `resumen_eda_preliminar.md` |

## 6. Conexión con Fase 3 — Preparación de datos / Matriz Madre

### 6.1 ¿Qué es la Matriz Madre?

La Matriz Madre es la tabla final que alimentará el análisis estadístico. Es una tabla donde:
- Cada fila = un país (o entidad comparable)
- Cada columna = un atributo (variable de cualquier fuente)
- Las celdas = valores reales auditados

### 6.2 ¿Qué necesita la Fase 3 que le da el EDA?

El EDA Preliminar genera el **mapa de navegación** para construir la Matriz Madre. Sin este EDA, la Fase 3 tendría que adivinar qué hay en los datos. Con este EDA, la Fase 3 tiene respuestas concretas:

```text
PREGUNTA DE FASE 3                          INPUT DEL EDA
──────────────────                          ─────────────

¿Qué variables incluir?          ──────→   inventario_variables.csv
                                            (sabe qué hay, qué tipos son,
                                             qué tan completos están)

¿Qué países entran a la muestra? ──────→   cobertura_pais_fuente.csv
                                            (sabe quién está en qué fuente,
                                             quién está en todas, quién es parcial)

¿Snapshot o panel?               ──────→   cobertura_temporal_fuente.csv
                                            (sabe qué fuentes son panel,
                                             cuáles son cross-section,
                                             qué años cubren)

¿Qué limpiar primero?            ──────→   data_quality_issues.csv
                                            (sabe qué problemas hay,
                                             su severidad, y qué fase los resuelve)

¿Cómo hacer las reglas de        ──────→   recomendaciones_wrangling.csv
  coalescencia?                            (sabe qué decisiones tomar,
                                             cuál es la opción recomendada,
                                             qué riesgo tiene ignorarlas)

¿Qué tabla de mapeo ISO3         ──────→   entidades_no_pais_o_sin_iso3.csv
  canónica usar?                           (sabe qué entidades no son países,
                                             qué hacer con EU, regiones, etc.)

¿Qué celdas están vacías?        ──────→   cobertura_pais_atributo.csv
  ¿Imputar o excluir?                     (sabe qué tan densa o sparse es
                                             cada combinación país-fuente)
```

## 7. Lo que el EDA NO hace (explícitamente)

Para evitar confusiones, el EDA Preliminar RAW NO:

- NO construye la Matriz Madre final
- NO decide cuál es el outcome (Y)
- NO decide cuál es el tratamiento regulatorio (X1)
- NO decide qué controles entran al modelo (X2)
- NO imputa valores faltantes
- NO codifica variables para modelamiento
- NO ejecuta regresiones ni tests de significancia
- NO interpreta si la regulación se asocia positiva o negativamente con el ecosistema IA
- NO decide la muestra analítica final
- NO elimina variables por baja cobertura
- NO crea índices compuestos

Estas decisiones pertenecen a las Fases 3-6 (Preparación, EDA principal, Feature Engineering, Modelado).

## 8. Riesgos conocidos y mitigaciones

| Riesgo | Descripción | Mitigación en el EDA |
|---|---|---|
| Mezclar EDA con modelamiento | Empezar a analizar asociaciones antes de auditar | Prohibir X/Y/modelos en esta fase |
| Sesgo de cobertura | OECD y Anthropic cubren pocos países | Reportar cobertura por fuente |
| Mezclar países con regiones | Stanford, Microsoft incluyen regiones/global | Clasificar entidades antes de integrar |
| Temporalidad inconsistente | Fuentes tienen años y escalas distintas | Registrar estructura, postergar snapshot |
| Variables redundantes | GII, WB, Oxford miden conceptos similares | Documentar redundancia, no eliminar |
| Rankings invertidos | En rankings menor puede ser mejor | Marcar is_rank_like y dirección pendiente |
| Datos sintéticos | Celdas inventadas o imputadas sin documentar | Auditar cada celda contra fuente original |

---

## 9. Criterio de verdad: cada dato debe ser auditable

Principio fundamental de este EDA: **cada dato, cada celda, cada estadístico debe ser trazable hasta su archivo fuente original.**

Esto significa:

1. Si el EDA dice que Chile tiene `ley_ia_vigente = 0`, un humano debe poder abrir `IAPP/iapp_dataset_unificado.xlsx`, ir a la hoja `datos`, buscar la fila de Chile, y verificar que la celda dice 0.

2. Si el EDA dice que Oxford tiene 195 países, un humano debe poder abrir `Oxford_Insights_Unificado.xlsx`, ir a la hoja `Consolidado`, y contar los ISO3 únicos.

3. Si el EDA dice que una columna tiene 60% de datos faltantes, un humano debe poder verificar ese porcentaje con los datos del archivo original.

**No se permite ningún dato sintético, inventado, extrapolado o imputado en esta fase.**
